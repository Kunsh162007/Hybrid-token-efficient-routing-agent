Hybrid Token-Efficient
Routing Agent

Every token you don't send is free accuracy.

Gemma (local, costs nothing) + Fireworks AI (remote, scored) — routed per task, in real time.

1

The scoring game

  • Leaderboard = remote token count + output accuracy
  • Local model tokens count as zero — unlimited free compute
  • So the optimal agent is not the smartest one; it is the one that knows when its free answer is already right
  • Naive remote-everything: pays for every task. Naive local-everything: fails the accuracy floor.
2

The escalation ladder

RUNGACTIONCOST
0classify + semantic cache lookupfree
1-2local Gemma attempt + verifier, reworded retryfree
3self-consistency vote (k samples, majority)free
4remote judge — 1-token YES/NO on the local winner~50-200 tok
5cheap remote, compressed prompt + local draft hintpaid
6strong remote — last resortpaid++

Exit at the first trustworthy rung. Most tasks never leave the free zone.

3

Trust you can't fake

  • Zero-token verifier — numeric answers must parse, MCQ letters must be among the options, Python must compile, summaries must honor stated length limits, sentiment needs label + justification
  • Self-consistency quorum — two independent local samples must agree before a free answer ships; contested votes pay 1 token for a remote verdict, not a full generation
  • Judge-gated categories — math word problems, code debugging, and logic never ship on local agreement alone; a judge-rejected answer can never win a vote
4

It gets cheaper as it runs

  • Adaptive thresholds — per-task-type success EMA lowers the local bar where Gemma keeps winning, raises it where it keeps escalating
  • Semantic answer cache — paid answers stored with MiniLM embeddings; near-duplicate queries are free forever after (verifier-guarded)
  • Learned router — logistic regression trained on our own eval logs predicts P(local succeeds); retrains in seconds on launch-day data
  • Budget guardrails — hard per-task token cap; agent ships its best free answer instead of overspending

Built to survive launch day

  • Models revealed at kickoff → ALLOWED_MODELS and FIREWORKS_BASE_URL are read from the environment at runtime; tiers picked by size, zero code changes
  • Harness contract built in: tasks.json → results.json, atomic writes, exit codes, 10-minute budget with per-task time caps
  • Containerized (python:3.11-slim, CPU-only, model baked in) — public linux/amd64 image, identical on a laptop and the scoring VM
  • 161 offline tests; clients mocked; graceful degradation everywhere
